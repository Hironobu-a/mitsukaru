**ステータス**: Draft

## 1. プロダクト概要

### 1.1 ビジョン

士業（税理士・弁護士・社会保険労務士・公認会計士・司法書士等）に特化した、日本最高品質の求人プラットフォームを構築する。既存の `tax.mitsukaru-pro.co.jp`（税理士特化）を技術的に刷新し、全士業への拡張を前提としたモダンアーキテクチャで再構築する。

### 1.2 ミッション

「士業の転職を、もっとスマートに。」

- 求職者：自分に最適な事務所・企業を効率的に見つけられる
- 採用企業：優秀な士業人材に確実にリーチできる
- プラットフォーム：士業転職市場のデファクトスタンダードとなる

### 1.3 既存プロダクトの課題（技術的負債）

| 課題 | 影響 | 新プロダクトでの解決策 |
|------|------|----------------------|
| Fat Controller | テスト不能・保守困難 | Service層 + FormRequest による責務分離 |
| Laravel 8 + PHP 8.0 | セキュリティ・パフォーマンス | Laravel 11 + PHP 8.3 |
| フロントエンド混在（Blade + jQuery） | UX低下・開発速度低下 | Next.js 15 による SPA/SSR ハイブリッド |
| テスト不足 | リグレッション頻発 | Pest + Playwright による自動テスト |
| 税理士のみ対応 | 事業拡大不可 | マスターテーブル設計による全士業対応 |
| SEO対策不十分 | 集客力不足 | SSR + 構造化データ + コンテンツ戦略 |

## 4. 非機能要件

### 4.1 パフォーマンス

| 指標 | 目標値 |
|------|--------|
| ページ初回表示（LCP） | 2.5秒以内 |
| インタラクション応答（FID） | 100ms以内 |
| レイアウトシフト（CLS） | 0.1以下 |
| API応答時間（p95） | 200ms以内 |
| 同時接続ユーザー数 | 1,000人以上 |
| 求人検索レスポンス | 100ms以内（キャッシュ時） |

### 4.2 セキュリティ

| 要件 | 対策 |
|------|------|
| 認証 | Laravel Sanctum（SPA認証）+ OAuth 2.0 |
| 認可 | Policy + Gate による RBAC |
| データ保護 | 個人情報の暗号化・SSL/TLS必須 |
| CSRF | SPA対応のCSRFトークン |
| XSS | React の自動エスケープ + CSP ヘッダー |
| SQLインジェクション | Eloquent ORM + Prepared Statement |
| レート制限 | API単位のスロットリング |
| 監査ログ | 重要操作のログ記録 |

### 4.3 可用性・信頼性

| 指標 | 目標値 |
|------|--------|
| 稼働率 | 99.9%（月間ダウンタイム43分以内） |
| RPO（目標復旧時点） | 1時間 |
| RTO（目標復旧時間） | 4時間 |
| バックアップ | 日次自動バックアップ + 7日間保持 |

### 4.4 スケーラビリティ

- 水平スケーリング対応（ステートレスAPI設計）
- CDN によるフロントエンド配信
- Redis によるキャッシュ・セッション管理
- キューワーカーによる非同期処理

### 4.5 アクセシビリティ

- WCAG 2.1 Level AA 準拠
- スクリーンリーダー対応
- キーボードナビゲーション対応
- カラーコントラスト比 4.5:1 以上

## 6. 制約条件

### 6.1 技術的制約

- 既存インフラ（AWS）との互換性を維持
- 既存の `tax.mitsukaru-pro.co.jp` からのデータ移行を考慮
- 少人数チーム（2-3名）での開発・運用を前提

### 6.2 ビジネス制約

- MVP ローンチまで3ヶ月以内
- 初期開発予算内での実現
- 既存顧客（税理士向け）のサービス継続

### 6.3 法的制約

- 個人情報保護法への準拠
- 職業安定法への準拠（求人広告の表示義務）
- 特定商取引法への準拠
